# AI-Powered Portfolio Template

## What This Project Is

This project is a portfolio template built with Next.js, Once UI, MDX, and the OpenAI Responses API. The main goal was to create a personal site that does more than statically list work. Visitors can explore projects, read posts, download a resume, and ask questions directly through a chat assistant.

## Why I Built It

Most portfolios are passive. They present information, but they do not help visitors navigate it. I wanted a portfolio that could:

1. explain my work clearly
2. answer follow-up questions about projects and experience
3. give visitors context about the page they are already viewing
4. demonstrate practical AI integration without adding unnecessary infrastructure

## What Makes It Useful

The chat assistant can:

- answer questions about experience, skills, and background
- reference blog posts and project pages
- understand the page the visitor is currently on
- stay grounded in actual portfolio content and resume context

That makes the site more useful for recruiters, hiring managers, collaborators, and clients who want a faster way to understand the work.

## Architecture

The implementation stays intentionally simple.

- **Next.js** handles rendering, routing, and API routes
- **Once UI** provides the component system and visual structure
- **MDX** powers editable blog and project content
- **OpenAI Responses API** powers the chat experience
- **Structured context assembly** keeps the assistant grounded without needing extra retrieval infrastructure

Instead of introducing extra retrieval infrastructure, the backend assembles context from the portfolio content itself, a resume context file, and the current page.

## Why That Approach Matters

For a personal portfolio, simplicity is a feature.

- Fewer moving parts means less maintenance
- Content updates stay in familiar files
- The architecture is easier to understand and explain
- Costs stay low
- The chat still feels informed and personalized

This is a better fit for a portfolio than adding infrastructure just because it is available.

## OpenAI Integration

The assistant uses the Responses API with multi-turn state. For follow-up turns, it chains with `previous_response_id` instead of resending the full history. That keeps the request shape clean and aligned with current OpenAI patterns.

The chat is also configured to stay concise by default, which makes it feel more like a useful portfolio assistant than a generic chatbot.

## What You Can Learn From It

This project is useful if you want to study:

- production-style Next.js API routes
- a clean frontend-to-backend chat flow
- grounded AI responses using local content
- streaming model responses in a simple UI
- practical architecture choices for low-cost AI features

## Why It Helps As A Portfolio

The project does more than present a list of technologies. It shows:

- full-stack implementation
- AI product integration
- practical system design choices
- attention to maintainability and operating cost

That makes it a stronger signal than a static portfolio alone.

## Final Thought

The point of this project was not to build the biggest AI stack possible. It was to build something sharp, useful, maintainable, and realistic. For a personal portfolio, that tradeoff matters.
